SESSION 1 — Initial Build
────────────────────────────────────────────────────────────────

PROJECT GOAL
  A Flask-based student face recognition attendance system with:
  - Anti-spoofing (liveness detection) to block photos/masks
  - Face recognition to identify enrolled students
  - Attendance logging with a web UI

ARCHITECTURE DECIDED
  Two separate models work together:
  1. Silent-Face-Anti-Spoofing (GitHub: minivision-ai)
       → Answers: "Is this a real face or a spoof?"
       → Must be cloned separately next to the project folder
  2. face_recognition library (pip install face_recognition)
       → Answers: "Whose face is this?"
       → Uses face_distance() for matching, not compare_faces()

RECOGNITION PIPELINE (confirmed by user)
  Camera frame
    → Silent-Face anti-spoof check   (block if fake)
    → face_recognition.face_encodings()
    → face_recognition.face_distance() vs all stored encodings
    → best match below threshold → mark present

DATABASE SCHEMA (original)
  Table: students   → id, name, encoding (JSON)
  Table: attendance → id, student_id, date, time

FILES CREATED
  app.py                         Flask backend + all API routes
  templates/base.html            Shared layout with navbar
  templates/index.html           Recognition page (camera + status)
  templates/enroll.html          Student enrollment page
  templates/attendance.html      Attendance log filtered by date
  static/css/main.css            Dark techy UI (Syne + DM Mono fonts)
  requirements.txt               flask, opencv-python, face_recognition,
                                 numpy, torch, torchvision
  README.md                      Setup instructions

API ROUTES (original)
  GET  /                         Recognition page
  GET  /enroll                   Enroll page
  GET  /attendance               Attendance page
  POST /api/enroll               Enroll student (name + image)
  POST /api/recognize            Recognize face + mark attendance
  GET  /api/attendance?date=X    Fetch attendance for a date
  GET  /api/students             List all students
  DELETE /api/students/<id>      Remove a student

SESSION 3 — Bug Fixes (anti-spoof error + attendance page)
────────────────────────────────────────────────────────────────

BUGS FIXED

  1. CRITICAL — Anti-spoof ufunc 'add' TypeError
     Error: "ufunc 'add' did not contain a loop with signature matching
             types (dtype('<U16'), dtype('uint8')) -> None"
     Root cause: Silent-Face's test() function expects a FILE PATH (string)
     as its first argument, not a numpy array. We were passing the decoded
     frame (np.ndarray) directly, which caused numpy to try to concatenate
     a string path with uint8 pixel data internally — hence the ufunc error.
     Fix (app.py): In is_real_face(), save the frame to a temp .jpg file
     using tempfile.NamedTemporaryFile + cv2.imwrite(), pass the file path
     to spoof_test(), then delete the temp file in the finally block.
     Files changed: app.py
       - Added `import tempfile`
       - is_real_face() now writes frame → tmp file → passes path → cleans up

  2. Attendance page stuck on "Loading…" when no students enrolled
     Root cause: The loadAttendance() async function had no try/catch.
     If the fetch threw any error (network hiccup, startup timing), the
     promise rejected silently and the tbody was never updated — leaving
     the hardcoded "Loading…" placeholder visible forever.
     Even when the fetch succeeded and returned [], the empty-state branch
     ran correctly — but any unhandled exception before that prevented it.
     Fix (attendance.html): Wrapped fetch in try/catch. On error, shows
     a descriptive message. On empty array [], correctly shows
     "No students enrolled yet." immediately.
     Files changed: templates/attendance.html
       - loadAttendance() now has try/catch around fetch
       - Error state renders a readable message in the table

SESSION 4 -- Bug Fixes (temp file path + old DB schema migration)
------------------------------------------------------------------------

BUGS FIXED

  1. Anti-spoof error: 'NoneType' object has no attribute 'shape'
     Root cause: The temp file path was NOT made absolute before os.chdir().
     When Silent-Face's test() internally called cv2.imread(tmp_path), it
     resolved the path relative to the NEW working directory (the Silent-Face
     repo folder), producing a corrupted path like:
       ./images/sample/C:\Users\...\AppData\Local\Temp\tmpXXX.jpg
     cv2.imread() returned None, and the next operation on it (.shape) crashed.
     Fix (app.py, is_real_face):
       tmp_path = os.path.abspath(tmp.name)   <-- added abspath() BEFORE chdir
     Also added a check: if cv2.imwrite() returns False, raise a clear error
     instead of silently passing None into the pipeline.

  2. sqlite3.OperationalError: no such column: week1
     Root cause: The database file already existed with the OLD schema
     (id, name, encoding only -- no week columns). SQLite's
     CREATE TABLE IF NOT EXISTS skips creation if the table exists,
     so the week columns were never added to the on-disk table.
     Fix (app.py, init_db):
       After CREATE TABLE IF NOT EXISTS, read the actual columns with
       PRAGMA table_info(students) and ALTER TABLE ADD COLUMN for any
       missing week1..week14 columns.
       This is a non-destructive migration -- existing student records
       and encodings are preserved, week columns are added with DEFAULT 0.
     Action required: just restart the app (python app.py). No need to
     delete the database manually.

FILES CHANGED
  app.py  -- is_real_face() absolute path fix + init_db() migration logic

SESSION 5 -- Root cause found and fixed: bypass test.py entirely
------------------------------------------------------------------------

ROOT CAUSE OF ALL ANTI-SPOOF ERRORS
  After reading the actual Silent-Face source code, the real problem was
  identified: test.py hardcodes a path prefix at the top of the file:
    SAMPLE_IMAGE_PATH = "./images/sample/"
  The test() function does:
    image = cv2.imread(SAMPLE_IMAGE_PATH + image_name)
  So no matter what path we pass in, it always prepends "./images/sample/".
  On Windows with an absolute path this produces:
    ./images/sample/C:\Users\...\Temp\tmpXXX.jpg
  which is an invalid path -- cv2.imread() returns None -- and then the
  very next line does image.shape which crashes with:
    'NoneType' object has no attribute 'shape'
  This cannot be fixed by making paths absolute. test.py is simply not
  designed to accept arbitrary paths -- it only works for files placed
  in its own ./images/sample/ folder.

FIX APPLIED
  Bypass test.py completely. Instead, call the underlying classes that
  test.py itself uses internally:
    from src.anti_spoof_predict import AntiSpoofPredict
    from src.generate_patches import CropImage
    from src.utility import parse_model_name
  These accept numpy arrays directly, no file paths involved at all.
  The logic mirrors what test.py does internally, but works with any
  numpy array passed from our Flask route.

  Additional improvement: models are loaded ONCE at startup via
  _load_spoof_models() instead of on every request. This makes
  the first scan much faster after the server starts.

  Removed tempfile import -- no longer needed at all.

DATABASE NOTE
  Migration code removed. init_db() is now simple again.
  Action required from user: delete database/attendance.db manually
  before restarting, since the old schema has no week columns.
  The file will be recreated fresh on next python app.py.

FILES CHANGED
  app.py -- complete rewrite of anti-spoofing integration

SESSION 6 -- Fixed parse_model_name unpack + missing get_bbox call
------------------------------------------------------------------------

BUGS FIXED

  1. "not enough values to unpack (expected 5, got 4)"
     Root cause: parse_model_name() returns exactly 4 values:
       (h_input, w_input, model_type, scale)
     parsed from filenames like: 2.7_80x80_MiniFASNetV2.pth
     We were unpacking 5 values which caused the ValueError immediately.
     Fix: changed unpack to exactly 4 variables.

  2. bbox=None was being passed to the image cropper
     In the previous version we skipped get_bbox() and passed bbox=None
     with crop=False. The correct flow (matching test.py exactly) is:
       image_bbox = _spoof_predictor.get_bbox(bgr_frame)
     and then pass that bbox with crop=True to the cropper.
     This is how the model was trained to receive input patches.
     Fix: added get_bbox() call before the model loop, crop=True in param.

  NOTE: Both fixes bring is_real_face() into exact alignment with test.py's
  internal loop, just operating on a numpy array instead of a file path.

FILES CHANGED
  app.py -- parse_model_name unpack fix + get_bbox() call added
